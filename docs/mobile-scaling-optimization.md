# 手機版遊戲元素縮放優化

## 問題描述
在調整畫布大小適應性後,手機版的遊戲元素(人物、掉落物)變得太小,影響遊戲體驗。

## 解決方案
引入 **動態縮放係數 (gameScale)** 系統,根據設備類型自動調整遊戲元素大小。

## 實施細節

### 1. 配置文件 (`game-config.js`)
新增配置項:
```javascript
MOBILE_SCALE_FACTOR: 1.8  // 手機版元素縮放係數(建議 1.5-2.0)
```

### 2. 遊戲邏輯 (`game.js`)

#### 初始化階段
- 根據螢幕寬度判斷設備類型
- 手機版: `gameScale = 1.8` (元素放大 1.8 倍)
- 桌機版: `gameScale = 1.0` (保持原始大小)

#### 應用範圍
1. **玩家角色**
   - 寬度/高度: `PLAYER.WIDTH/HEIGHT * gameScale`
   - Y 軸偏移: `PLAYER.Y_OFFSET * gameScale`
   
2. **掉落物品**
   - 尺寸: `ITEM_DEFAULT_SIZE * gameScale`
   - 生成位置: `ITEM_SPAWN_Y_OFFSET * gameScale`

3. **響應式調整**
   - 視窗大小變化時自動更新 `gameScale`
   - 重新計算玩家位置和尺寸

## 效果對比

### 調整前
- 手機版畫布高度: 1200px
- 玩家大小: 100x100px (固定)
- 掉落物大小: 60x60px (固定)
- **問題**: 元素相對畫布太小

### 調整後
- 手機版畫布高度: 1200px
- 玩家大小: 180x180px (100 × 1.8)
- 掉落物大小: 108x108px (60 × 1.8)
- **改善**: 元素大小適中,易於操作

## 自訂調整

如需調整縮放比例,修改 `game-config.js`:

```javascript
MOBILE_SCALE_FACTOR: 1.5  // 較小的放大倍率
MOBILE_SCALE_FACTOR: 2.0  // 較大的放大倍率
```

**建議範圍**: 1.5 - 2.0

## 技術優勢
✅ 桌機版完全不受影響  
✅ 自動響應視窗大小變化  
✅ 統一管理縮放係數  
✅ 易於調整和維護
